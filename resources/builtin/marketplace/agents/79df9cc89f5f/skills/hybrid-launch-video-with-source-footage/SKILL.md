## Procedure

1. **Collect brand constraints**
   - ALWAYS inspect the website and supplied assets before writing concepts.
   - ALWAYS restate required tone, color direction, language, duration, aspect ratio, subtitles, and must-use footage.

2. **Gate A: concepts**
   - ALWAYS offer 2–3 distinct concepts with hook, visual style, message, duration, and best use case.
   - WHEN the user has supplied a mascot/logo/footage, THEN make at least one concept use it as the opening hook.

3. **Gate B: script + EDL**
   - ALWAYS write a timed timeline with scene durations, visual action, subtitles/VO, and asset usage.
   - WHEN footage must appear, THEN encode it as `source_required` / delivery promise in the plan.
   - ALWAYS validate the plan and run the anti-slideshow / motion-ratio promise check before presenting the plan.

4. **Draft assembly**
   - ALWAYS keep subtitles readable and short; place them as native overlays tied to timeline beats.
   - ALWAYS include the supplied footage in the rendered timeline, not only as a reference frame.
   - WHEN using TTS, THEN generate narration to fit the target duration; shorten copy if it would require rushed speech.

5. **Audio sync QA**
   - NEVER deliver a draft with duplicate narration/music creating two voices.
   - WHEN narration sync matters, THEN split the script by scene beats and mix each segment at its exact timeline offset.
   - ALWAYS verify single voice track, coverage across intended scenes, and acceptable CTA/music-only tail before presenting.

6. **Delivery report**
   - ALWAYS report duration, resolution/aspect ratio, audio status, subtitle/readability status, source-footage inclusion, and motion-ratio/promise-check result.
   - WHEN there is a known gap such as a silent CTA tail, THEN state it explicitly as acceptable or ask whether to change it.
